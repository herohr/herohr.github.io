J2EE初窥（一）

HeroHR 发表于 2019-02-17    标签： J2EE

Java EE

Java重量级开发平台Java Enterprise Edition主要面向于服务端，提供了一整套完备的、工程化的开发框架，为开发大中型服务端应用提供了整套解决方案。

Java EE框架组件包含了web技术的jsp、servlet、Structs2，业务层的EJava Bean，持久化层的MyBatis、JPA等。

Web技术

JSP

Java Server Page是一种以HTML为主，中间穿插Java代码的服务端语言。JSP的创造主要是为了解决视图层中页面逻辑与业务逻辑纠缠的问题，将HTML渲染放到服务器端来做。

随着前端技术的不断发展，前后端分离的架构已经成为主流，JSP难以符合软件工程的要求，JSP渐渐已经退出Java Web技术主流，仅在政企及学校继续苟延残喘。

Servlet

Servlet与容器

Servlet作为Java EE的代表框架，撑起了Java 服务端技术的半边天。Servlet是服务端小程序，一个Servlet仅仅是一个类，而他则依赖于其背后的Servlet框架及web容器。流行的容器比如Tomcat，它将HTTP请求进行处理和预包装，向下传递给Servlet，Servlet进行业务逻辑处理，向上传递给Tomcat，Tomcat再将响应回送给客户端。这种架构很像Apache与PHP、Gunicorn与Django（DaiAnGou）.

Servlet结构与生命周期

Servlet类有init方法、destroy方法，分别在Servlet实例被调用时及被销毁时调用。

Servlet类有service方法，当一个请求到达时（无论method如何）则会将请求传递给service方法，进行处理。但是，一般情况下并不重写service方法，因为service方法还会继续传递请求给doGet、doPost、doXXX方法继续处理。doGet等方法在请求属于对应method的情况下调用。

如上图，当第一个符合该servlet应服务的请求到达时，才将该servlet的init方法调用，之后的请求到达时，不再调用该方法，直接进入service方法处理。当受到终止服务时，调用destroy方法。

Filter过滤器

用户验证、表单验证、编码转换等对于请求的预处理在许多处理逻辑中都具有，Filter则将其解耦。Filter的设计核心是职责链模式，对于某个请求，需要进行多步过滤，为了让每一个过滤器都有机会过滤，将各个过滤器按顺序排为过滤器链(FilterChain)。

Filter不仅仅具有前过滤，还拥有后过滤，也就是在其他下级过滤器或Servlet处理完之后再进行过滤，以达到重新编码、Cookie设置等操作。

（国标上要求国内网站都要以GBK编码，但一般程序却善于处理UTF编码，故编解码问题是一个令人头疼的问题，编写一个编解码过滤器，将会巧妙解决该问题）

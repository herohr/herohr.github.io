震惊，Python中竟然有这种史诗巨坑

HeroHR 发表于 2017-03-01    标签： python

OUT : [2, 2, 1, 1]

这是怎么回事？

按照Python工程师的逻辑，这种bug不可能会出现

for循环内部使用一个列表的下标进行遍历，每循环一遍，下标自增

用一个不健壮的c语言代码描述，就是这样的，当remove之后，被remove掉的元素后面的元素全部前移，但是下标仍然自增，这样就会跳过一个元素。

所以会出现上述情况。

这样的写法也许会更好，虽然这样并不pythonic

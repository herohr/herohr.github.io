J2EE初窥（一）

HeroHR 发表于 2019-02-17    标签： J2EE

Java EE

Java重量级开发平台Java Enterprise Edition主要面向于服务端，提供了一整套完备的、工程化的开发框架，为开发大中型服务端应用提供了整套解决方案。

Java EE框架组件包含了web技术的jsp、servlet、Structs2，业务层的EJava Bean，持久化层的MyBatis、JPA等。

Web技术

JSP

Java Server Page是一种以HTML为主，中间穿插Java代码的服务端语言。JSP的创造主要是为了解决视图层中页面逻辑与业务逻辑纠缠的问题，将HTML渲染放到服务器端来做。

随着前端技术的不断发展，前后端分离的架构已经成为主流，JSP难以符合软件工程的要求，JSP渐渐已经退出Java Web技术主流，仅在政企及学校继续苟延残喘。

Servlet

Servlet与容器

Servlet作为Java EE的代表框架，撑起了Java 服务端技术的半边天。Servlet是服务端小程序，一个Servlet仅仅是一个类，而他则依赖于其背后的Servlet框架及web容器。流行的容器比如Tomcat，它将HTTP请求进行处理和预包装，向下传递给Servlet，Servlet进行业务逻辑处理，向上传递给Tomcat，Tomcat再将响应回送给客户端。这种架构很像Apache与PHP、Gunicorn与Django（DaiAnGou）.

Servlet结构与生命周期

Servlet类有init方法、destroy方法，分别在Servlet实例被调用时及被销毁时调用。

Servlet类有service方法，当一个请求到达时（无论method如何）则会将请求传递给service方法，进行处理。但是，一般情况下并不重写service方法，因为service方法还会继续传递请求给doGet、doPost、doXXX方法继续处理。doGet等方法在请求属于对应method的情况下调用。

如上图，当第一个符合该servlet应服务的请求到达时，才将该servlet的init方法调用，之后的请求到达时，不再调用该方法，直接进入service方法处理。当受到终止服务时，调用destroy方法。

Filter过滤器

用户验证、表单验证、编码转换等对于请求的预处理在许多处理逻辑中都具有，Filter则将其解耦。Filter的设计核心是职责链模式，对于某个请求，需要进行多步过滤，为了让每一个过滤器都有机会过滤，将各个过滤器按顺序排为过滤器链(FilterChain)。

Filter不仅仅具有前过滤，还拥有后过滤，也就是在其他下级过滤器或Servlet处理完之后再进行过滤，以达到重新编码、Cookie设置等操作。

（国标上要求国内网站都要以GBK编码，但一般程序却善于处理UTF编码，故编解码问题是一个令人头疼的问题，编写一个编解码过滤器，将会巧妙解决该问题）

Python并发编程（三）——乐观锁与悲观锁

HeroHR 发表于 2017-11-27    标签： server / python

悲观锁

互斥锁，可重入锁，自旋锁等都属于悲观锁。

当你读取或者写入数据时，你都悲观地认为有别人正在改动数据，所以你希望在你 操作数据的时候上锁，防止别人改动。当然，当已经有别人上锁，你必须等待别人操作完毕。

那么，由此而来的等待时间，在某些场景下影响了性能。

乐观锁

乐观锁以版本号作为分辨更新有效的手段。

当你读取数据的时候，同时读取一个版本号，当你准备以刚刚读取的数据为基础来更新数据时，需要对比刚刚的版本号与现在的版本号是否一致，如果一致，则版本号自增，如果不一致，说明有其他人已经修改过数据了，那么你这次的修改就将放弃。

乐观锁避免了“长事务”下读写的性能问题。

在并发程序中，多个线程同时操作一个资源的时候，有可能发生不可预料的异常。

举个例子，获取某个资源的值，并在此基础上自增1。

我们创建两个线程，两个线程都拥有一个a变量。每个线程都进行自增操作20000次，如果a的初始值为0，那么两个线程工作完之后a的值应该是40000。（for 循环为了 加长操作的时间）

但是注意，两个线程是同时执行的，如果在线程1执行完b=a之后放弃了运行权，线程2此时执行b=a，并执行自增操作，之后线程1被唤醒，将以前的值自增，赋给a。这时候线程2的操作相当于无效了。

此时，需要一个能够让两个线程相互了解对方的状态，并在对方操作某些资源把自己阻塞，等待对方操作完毕。

threading.Lock 对象 قفل

这个对象只有两种方法，lock.acquire(), lock.release()

Lock是一把锁子，这个锁不对别的资源加锁，而是对自己加锁。Lock对象加锁的时候，其他地方如果尝试加锁，则会阻塞（等待）直到Lock对象解锁。

首先使用Lock() 创建一个锁， 在某个线程操作某个公共资源时候，lock.acquire()获取锁，并在操作完毕lock.release()。这时候其他的线程的lock.acquire()调用就停止了阻塞状态，该线程获取了锁，然后操作公共资源。

当操作完公共资源而忘记解锁时，则会出现死锁情况，这时候将会一直阻塞，一定要避免死锁的出现。

Python并发编程（一）——线程池

HeroHR 发表于 2017-10-16    标签： server / python

Python并发库concurrent.futures

Python的多线程与多进程库threading和multiprocess库提供了简单的并发封装。但是我们在高并发的情况下，往往需要限制线程数和进程数。concurrent.futures库提供了非常好用的线程进程池，我认为是可以代替threading库和multiprocess库了！

基类 Executor

线程池和进程池都继承于这个类，所以提供了一套通用的方法去提交futures。

submit(fn, *args, **kwargs)

这个方法必须接受一个函数为参数，并在后面接收该函数的参数。如果有一个sum(a, b, float=True）的函数需要异步执行，那么只需要executor.submit(sum, a, b, float=True)如此调用，即可将此异步事件放入线程池执行。

这个方法返回一个 future对象。

map(fn, *iterables)

这个方法必须接受一个函数为参数，然后将后面的可迭代参数内容进行打包，变成函数的参数。如果有5个sum函数需要放入线程池执行，那么map(sum, (1, 2, 3, 4,), (5, 6, 7, 8))即等于submit(sum, 1, 5), submit(sum, 2, 5)….四个submit。

这个方法返回一个future列表。

shutdown()

关闭线程池。当使用上下文管理器时，不用显式调用这个方法，上下文管理器会自己关闭线程池。

线程池的两种创建方式

executor = ThreadPoolExecutor(max_workers=x)

这个函数可以创建一个线程池，最大线程数等于x。

with ThreadPoolexecutor(max_workers=x):

这个上下文管理器可以当你不用的时候调用shutdown()方法，关掉线程池。

Future 类

上文提到了submit方法会返回一个future对象，future对象是Future类的实例。通过future对象，我们可以知道我们的异步事件执行到何种程度，可以拿到函数的返回值。

cancle()

取消调用，如果已经执行中且无法终止，会返回False，如果可以取消则返回True。

cancelled()

查看是否被取消。

running()

查看是否正在运行。

done()

查看是否完成。

result（timeout=None）

这个方法比较重要，返回函数的调用结果。当没有设置timeout时，将一直等到函数执行完毕，无论这个函数有没有被执行。当设置timeout之后，超时则会引发timeout错误。

exception(timeout=None)

获取函数的错误。

add_down_callback(fn)

给future添加回调函数，回调函数应该仅接受一个参数 future，当future完成后，将会调用这个回调函数，并将future对象传入。

as_complete(future_list, timeout=None)函数

模块函数，接受future列表。是一个生成器，每当有一个future完成，将生成这个future对象。一般是用做for循环的迭代器的。

一旦futures里有future完成，将会进入一次循环，并打印result值。

Go中的结构体方法与接口

HeroHR 发表于 2017-08-02    标签： golang

Go语言中的对象用结构体来表现，而对象的方法则是用函数来表现。

定义结构体

和C语言非常相似

定义方法

值传递方法

值传递方法，即进入函数调用栈时再次为结构体划分内存，然后复制结构体。在值传递方法内修改属性，并不改变外部结构体的属性。

引用传递方法

引用传递方法，即进入函数调用栈时，仅仅传递一个引用，对结构体的操作都将影响外部结构体。

总的来说，一般只读方法可以使用值传递，写方法使用引用传递。

定义接口

实现了某些方法的对象，他们可以抽象出来一个接口。

正方形和圆形都实现了area()的方法，所以可以定义一个几何体的接口。q实现了area方法的结构体都可以使用这个接口。
